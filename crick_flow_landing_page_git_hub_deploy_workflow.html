# Repository: crickflow-landing This single file contains all assets for a lightweight, accessible, production-ready static landing page for **CrickFlow** plus a GitHub Actions workflow to auto-deploy to GitHub Pages. --- ## Files included (each below is a separate file you can copy into your repo) ### 1) `README.md` ```md # CrickFlow — Landing Page This repository contains a static landing page for **CrickFlow** — a fictional cricket analytics / fan engagement product. It is lightweight, responsive, and accessible. ## Structure ``` / (root) ├─ index.html ├─ assets/ │ ├─ css/styles.css │ └─ js/main.js ├─ images/ │ └─ (place hero/team logos if desired) └─ .github/workflows/pages.yml ``` ## Deploy 1. Push this repo to GitHub (e.g. `crickflow`). 2. Enable GitHub Pages (deploy from `gh-pages` branch or `main`/`docs/` depending on workflow). The included workflow deploys automatically to GitHub Pages using `peaceiris/actions-gh-pages`. ## License MIT ``` ``` --- ### 2) `.github/workflows/pages.yml` (GitHub Actions — deploy to GitHub Pages) ```yaml name: Deploy to GitHub Pages on: push: branches: [ main ] permissions: contents: write jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 with: persist-credentials: false - name: Setup Node uses: actions/setup-node@v4 with: node-version: '18' - name: Install and build (no-op for static) run: | # If you add a build step (webpack, hugo, etc.) add it here. echo "No build step." - name: Deploy to gh-pages uses: peaceiris/actions-gh-pages@v3 with: github_token: ${{ secrets.GITHUB_TOKEN }} publish_dir: ./ publish_branch: gh-pages ``` > This workflow publishes the repository root to the `gh-pages` branch whenever you push to `main`. GitHub Pages should be configured to serve from the `gh-pages` branch. --- ### 3) `index.html` ```html CrickFlow — Real-time cricket analytics & fan tools
